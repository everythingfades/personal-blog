    \(\begin{array}{c|c} O(1) & O(n)\\ cons & snoc\\ head & last\\ tail & init \end{array}\)

    we want all of those to be \(O(1)\)

    Deque

    data Deque a = Deque Int [a] [a] derving show

    given x = [1,2,3]

    Haskell
    head :: Deque a -> a
head (Deque _ [] [v]) = v -- O(1)
head (Deque _ (u:us) _) = u -- O(1)

last :: Deque a -> a
last (Deque _ [v] []) = v -- O(1)

    symmetrical

    Haskell
    cons :: a -> Deque a -> Deque a -- O(1)
cons u (Deque n sv []) = Deque (n+1) [u] sv
cons u (Deque n us sv) = Deque (n+1) (u:us) sv

snoc :: Deque a -> a -> Deque a -- O(1)
snoc (Deque n [] us) v = Deque (n+1) us [v]
snoc (Deque n us sv) v = Deque (n+1) us (v:sv)

    now the head, last, cons, snoc are all O(1)

    Haskell
    tail :: Deque a -> Deque a -- O(n) in worst case, ~O(1)
tail (Deque 0 _ _) = undefined -- O(1)
tail (Deque 1 _ _) = nil -- O(1) -- we dont care what is inside, there is only one element in the Deque
-- tail (Deque n [u] sv) = Deque (n-1) [] sv -- this is illegal, possibly break the invariance
tail (Deque _ [_] sv) = fromList (reverse sv) -- O(n)
tail (Deque _ (_:us) sv) = Deque (n-1) us sv

init :: Deque a -> Deque a -- O(n) in worst case, ~O(1)
init (Deque 0 _ _) = undefined
init (Deque 1 _ _) = nil
init (Deque _ us [_] ) = fromList us
init (Deque n us (_:sv)) = Deque (n-1) us sv

    analytically, the functions is O(n), but this is not the case

    Sadly, it appears as if, though symmetric, the functions init and tail are expensive as O(n). In reality, we are going to learn that this is NOT the case (see the associated .pdf). You might also be able to see why by looking at the following sequence of 16 tails.

    Haskell
    [ Deque 16 [1] [16,15,14,13,12,11,10,9,8,7,6,5,4,3,2] -- O(n)
, Deque 15 [2,3,4,5,6,7,8] [16,15,14,13,12,11,10,9]   -- O(1)
, Deque 14 [3,4,5,6,7,8] [16,15,14,13,12,11,10,9],    -- O(1)
, Deque 13 [4,5,6,7,8] [16,15,14,13,12,11,10,9],      -- O(1)
, Deque 12 [5,6,7,8] [16,15,14,13,12,11,10,9],        -- O(1)
, Deque 11 [6,7,8] [16,15,14,13,12,11,10,9],          -- O(1)
, Deque 10 [7,8] [16,15,14,13,12,11,10,9],            -- O(1)
, Deque 9 [8] [16,15,14,13,12,11,10,9],               -- O(n/2)
, Deque 8 [9,10,11,12] [16,15,14,13]                  -- O(1)
, Deque 7 [10,11,12] [16,15,14,13]                    -- O(1)
, Deque 6 [11,12] [16,15,14,13],                      -- O(1)
, Deque 5 [12] [16,15,14,13],                         -- O(n/4)
, Deque 4 [13,14] [16,15],                            -- O(1)
, Deque 3 [14] [16,15],                               -- O(n/8)
, Deque 2 [15] [16],                                  -- O(1)
, Deque 1 [] [16],                                    -- O(1)
, Deque 0 [] []
]

    if we sum up these we get \(n + \frac{n}{2} + \frac{n}{4} + \dots = 2n\) then some \(O(1)\) s, in total, O(3n), so tail is O(3) = O(1)

    so the amortization illustates that we overpay and pay back we we have to do some extra operation(in this case, the rebalancing)

    amortizaed analysis

    we need three things

    • A cost function \(C_{op_i}(xs_i)\), this is the true cost of the operation opi st the state xsi
    • An amortized cost function \(A_{op_i}(xs_i)\) this is the amortised cost of opi when applied to xsi
    • A "potential" function (assume we are scientists), \(\Phi (xs)\) a measure of how expensive the next operation will be, >= 0

    Basically, these three functions will form a relationship

    \(C_{opi}(xs_i) \le A_{op_i}(xs) - (\Phi(xs') - Phi(xs))\)

    if this relationship is true, then the operation wil have complexity \(A_{op}\)

    summing up the both sides gives

    \(\sum_{i=0}^n C_{op_i}(xs_i) \le \sum_{i=0} A_{op_i}(xs_i) (\text{here, all the }\Phi\text{cancelles out})\)

    so

    \(\sum_{i=0}^n C_{op_i}(xs_i) + \Phi(xs) \le \sum_{i=0} A_{op_i}(xs_i) (\text{here, all the }\Phi\text{cancelles out})\)

    since

    \(\Phi(xs) \ge 0\) as we define it

    Re: analysis

    Haskell
    tail :: Deque a -> Deque a -- ~O(1)
tail (Deque 0 _ _) = undefined -- O(1)
tail (Deque 1 _ _) = nil -- O(1)
tail (Deque _ [_] sv) = fromList (reverse sv) -- O(n)
tail (Deque _ (_:us) sv) = Deque (n-1) us sv -- O(1)

    \(\begin{aligned} C_{tail}(\text{Deque 1 _ _}) &= 1\\ C_{tail}(\text{Deque _ [_] _}) &= n\\ C_{tail}(\text{Deque _ (_:_) _}) &= 1 \end{aligned}\)

    and

    the worst case for tail is linear time

    \(\begin{aligned} A_{tail}(xs) = k &\text{we assume this is } O(1)\\ \end{aligned}\)

    many \(\Phi\) can work, but we will use this

    \(\Phi(\text{Deque _ us sv}) = \max((\text{length sv} - \text{length us}), 0)\)

    so

    \(C_{op_i} \le A_{op_i} - (\Phi(xs') - \Phi(xs_i))\)

    so when length of us is 1 in Deque n us sv:

    \(\begin{aligned} C_{tail}(\text{Deque 1 _ _})&\le A_{tail}(\text{Deque 1 _ _}) - (\Phi(nil) - \Phi(\text{Deque 1 _ _}))\\ 1 &\le A_{tail}(\text{Deque 1 _ _}) - (\Phi(nil) - \Phi(\text{Deque 1 _ _}))\\ 1 &\le k - (0 - \Phi(\text{Deque 1 _ _}))\\ &\text{(in the worst case we end up with 0 as the potential of the singleton deque)}\\ 1 &\le k - (0 - 0)\\ 1 &\le k \end{aligned}\)

    so k must be greater than 1

    now we try to find what is going to happen when length us >= 2

    \(\begin{aligned} C_{tail}(\text{Deque n (_:us) sv}) &\le A_{tail}(\text{Deque n (_:us) sv})\\ 1 & \le k - (\Phi(\text{Deque (n-1) us sv}) - \Phi(\text{Deque n (_:us) sv}))\\ 1 &\le k - (\text{max (length sv - length us) 0} - \text{max (length sv - length (_:us)) 0}) \end{aligned}\)

    suppose us is longer than sv, then this gives \(1\le k\)

    else assum then length is m and n

    \(\begin{aligned} 1&\le k-((m-n) - (m- (n+1)))\\ 1 &\le k-(m-n -m +n + 1)\\ 1 &\le k-1\\ 2 &\le k \end{aligned}\)

    so we need \(~O(2)\) at least, we consider the last case, where us is a singleton list [_]

    \(\begin{aligned} C_{tail}(\text{Deque n [_] sv})&\le k- (\Phi(\text{fromList (reverse sv)}) - \Phi(\text{Deque n [_] sv}))\\ n-1 &\le k- (\Phi(\text{fromList (reverse sv)}) - \Phi(\text{Deque n [_] sv}))\\ n-1 &\le k - (1 - \Phi(\text{Deque n [_] sv}))\\ n-1 & \le k - (1-(n-2))\\ n-1 & \le k + n - 3\\ 2 & \le k \end{aligned}\)

    so all the three equation is true when k = 2, so it gives us a result of tail is O(2), the amortise analysis is based on balanced lists, and assumes that \(\Phi(xs_0) = 0\), if start from this tail is still O(2), so the amortise complexity of tail is O(1)

    exercise reverse = ~O(1)

    To "do" this, you've picked A_reverse(_) = k, so find a Phi(xs) such that Phi(reverse xs) - Phi(xs) = C_reverse(xs) = n. This would be the necessary requirement for your proof; you should realise pretty quickly that because reverses never have "expensive" and "cheap" cases, it's impossible to find a Phi function that can end up having this difference. That should help you realise that the Phi isn't just some "hack", it actually only works when there is a meaningful difference between states, and that the operation does vary in its costs throughout.
